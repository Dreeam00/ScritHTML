Technical リファレンス

ScritHTML v2.0 "Pragmatic" のすべてのAPI、ディレクティブ、設計パターンについて解説します。

Manifest

導入ガイド

ScritHTMLを使い始めるのは非常に簡単です。ビルドステップは必要ありません。CDNから scrit.js を読み込むだけで、HTMLがリアクティブなアプリケーションに進化します。 

<script src="scrit.js"></script>
<body>
  <store name="app">
    <value name="count">0</value>
  </store>
  <h1>{app.count}</h1>
</body>

ScritHTMLタグを直接 <body> 内に記述するだけで、自動的に解析され、DOMとして描画されます。

ディレクティブ API

v2.0では、新たに HTML タグベースのディレクティブを採用しました。これにより、標準的なHTMLとしての外見を保ちつつ、直感的にロジックを埋め込むことが可能です。 

<if condition="isLogined">
  <p>ようこそ！</p>
</if>

<for @each={<$item in="items">}>
  <li>{item}</li>
</for>

@ プレフィックスによるイベントリスナー（例: @click）や、{expression} による補間も強力な機能です。

グローバル・ステート

状態管理は <store> タグで行います。作成されたストアは、アプリケーション内のどこからでもアクセス可能なリアクティブなオブジェクトになります。 

<store name="user">
  <value name="name">"John"</value>
  <value name="age">25</value>
</store>

<p>名前: {user.name}</p>

ストアの値が変更されると、その値を参照している箇所が自動的に、そして最小限のコストで更新されます。

コンポーネント設計

再利用可能な部品は <component> タグで定義します。 <slot> を使うことで、親から渡されたコンテンツを任意の場所に挿入することも可能です。 

<component name="Card">
  <div class="card">
    <h3>{title}</h3>
    <slot></slot>
  </div>
</component>
